\subsection{Error Analysis}\label{sec:results-errors}

Confusion concentrates among visually similar class pairs (Fig.~\ref{fig:confusion}). The equipment/absence pairs (helmet vs.\ no\_helmet, goggles vs.\ no\_goggle) are separated by fine-grained head-region cues that degrade with distance and partial occlusion; \phX\% of no\_helmet errors fall into this pattern. Dark headwear and hoods account for a further \phX\% of helmet-related confusions, and gloves at long range are confused with bare hands in \phX\% of cases.

\subsection{Runtime and Deployment Feasibility}\label{sec:results-runtime}

Single-frame latency on a Tesla T4-class GPU is \phX~ms end to end: \phX~ms preprocessing, \phX~ms inference, and \phX~ms postprocessing at $640\times640$ resolution. At an analysis rate of \phX frames per second per camera, one GPU of this class sustains approximately \phX concurrent camera feeds, before any batching across streams.

The model is small (3.0M parameters, 8.1 GFLOPs, a 6.3~MB weight file \cite{ultralytics2023}), in line with edge-oriented PPE deployments \cite{edgeyolo2024}, so the detection workload for a multi-camera installation concentrates on one commodity GPU server on the existing network, with no per-camera hardware.
